Why Alibaba Cloud

The model and the metal, from one provider, in one region — low latency, no egress surprises.

Qwen Cloud is the brain

Every call — decomposition, curiosity, consolidation, embeddings, the answer — goes to Qwen models on the Model Studio OpenAI-compatible endpoint. qwen-max + text-embedding-v3.

ECS runs the substrate

The Rust kiokud daemon owns a 1 TiB sparse vRAM universe and a 4 TiB virtual disk — committed only on touch. Small outside, huge inside.

One region, one hop

Put the ECS instance in the same region as your Model Studio endpoint and the brain is a millisecond away. Substrate state lives on a persistent volume.

Deploy in 6 steps

From zero to a public arena. Full detail in PROOF.md.

  1. Create an ECS instance — Ubuntu 22.04, ≥ 2 GiB RAM (e.g. ecs.t6-c1m2.large), with a public IP.
  2. Open the security group — allow inbound TCP 8000 (app) and 22 (SSH).
  3. Install Docker on the instance: curl -fsSL https://get.docker.com | sh
  4. Get a Qwen key from Model Studio and put it in .env as QWEN_API_KEY.
  5. Ship it from your machine: 
    # builds the image (Rust kiokud + engine + arena) and runs it on the instance
./deploy/alibaba/deploy.sh ecs ubuntu@<public-ip>
  6. Verify the substrate is live on Alibaba Cloud: 
    curl http://<public-ip>:8000/api/health
→ {"ok":true,"backend":"kiokud","version":"0.1.0"}
